Boston MBTA Real-Time Dashboard

Python Apache Airflow PostgreSQL AWS RDS AWS EC2 Astronomer Streamlit Plotly GitHub Actions

Boston MBTA Real-Time Dashboard is a production-grade, real-time ETL pipeline that pulls live transit data from the MBTA V3 API every 2 minutes, orchestrated by Apache Airflow on Astronomer, stored in PostgreSQL on AWS RDS, and visualized in an interactive Streamlit dashboard.

What I built

  • Engineered an end-to-end Extract → Transform → Load pipeline collecting three live data streams — arrival/departure predictions (~2,200/run), vehicle positions (~114/run), and service alerts — with automatic retries and exponential backoff on the MBTA V3 API.
  • Orchestrated the pipeline as an Airflow 3.2.2 DAG (TaskFlow API) deployed on Astronomer, running every 2 minutes across five sequential tasks — schema init, extract, transform, bulk load with ON CONFLICT DO NOTHING, and rolling 30-day cleanup — with an audit table logging every run. 16M+ predictions ingested to date.
  • Deployed cloud infrastructure on AWS — PostgreSQL 15 on RDS and a t3.micro EC2 instance (us-east-2) — with a normalized schema indexed on route and timestamp.
  • Built an interactive Streamlit + Plotly dashboard with a live vehicle map (decoded route polylines, official MBTA route colors), KPI cards (on-time rate, active vehicles, alerts), route/direction/stop/time filters, stop drill-down for the next 5 arrivals, and 120-second auto-refresh.
  • Enforced quality with 65 unit tests across API client, transformer, and DB loader, gated by a GitHub Actions CI pipeline with a 70% coverage threshold on every push.
