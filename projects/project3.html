DineTrack

MySQL GCP Cloud SQL R DBI RMarkdown ggplot2

Restaurant visit analytics on Cloud SQL.

DineTrack simulates a real-world restaurant reporting pipeline: a normalized relational database hosted on Google Cloud SQL, structured data loaded via R, and a full analytics report generated in RMarkdown — surfacing customer behavior, food-vs-alcohol revenue trends, and location-level sales performance.

What I built

  • Designed a 3NF-normalized schema with dimension tables (customer, product, location, time, business-unit) and fact tables (visit, yearly sales, customer) in MySQL 8 on GCP Cloud SQL.
  • Scripted the full lifecycle in R — schema creation, CSV loading via DBI/RMySQL, stored-procedure business logic, referential-integrity validation, and teardown.
  • Authored analytical SQL for revenue, party size, and loyalty trends, and rendered a polished report with ggplot2, kableExtra, and RMarkdown (PDF/HTML).
